# دليل المشرف الأعلى — وحدة تحكّم المنصّة

**الأدوار:** المشرف الأعلى (`super_admin`) / مشرف المنصّة (`platform_admin`).
**التطبيق:** وحدة تحكّم المنصّة (`platform-web`) — واجهة داكنة «Mission Control».

هذه الوحدة **غير مرتبطة بقبيلة** (NOT tenant-scoped) ومحميّة بحارس `SuperAdminGuard`. الدخول حصريّ لمن `isSuperAdmin`؛ وأيّ مستخدم آخر يُرفَض عند الحدّ وتُغلَق جلسته. راجع [دليل البدء](./00-getting-started.md) لتسجيل الدخول.

بنية الواجهة: شريط علويّ داكن بثلاثة تبويبات — **اللوحة**، **القبائل**، **الإحصاءات** — وزرّ لغة وخروج.

---

## 1. اللوحة — `/dashboard`

> **وصف الشاشة:** عنوان «اللوحة» وزرّ **تحديث**. ثلاث بطاقات كبيرة بأيقونات ملوّنة: **عدد القبائل**، **إجمالي الأشخاص**، **إجمالي المستخدمين**. أرقام كبيرة مع حالة تحميل «—».
>
> `> [لقطة: لوحة المنصّة السريعة — تُضاف عند النشر]`

- المصدر: `GET /platform/stats` ← `{ tribes, persons, users }`.

---

## 2. القبائل (المستأجرون) — `/tenants`

> **وصف الشاشة:** عنوان «القبائل» ووصف، وزرّ **+ إنشاء قبيلة**. جدول بأعمدة: **الاسم** (مع شارة انتهاء اشتراك كهرمانيّة/حمراء عند الاقتراب/التجاوز)، **المُعرّف (slug)**، **عدد الأشخاص**، **عدد المستخدمين**، **الحالة** (وسم نشِطة/موقوفة)، وعمود **إجراءات**: زرّ **إدارة الاشتراك**، وزرّ **إيقاف** (للنشِطة) أو **تفعيل** (للموقوفة). الصفوف قرب الانتهاء مُظلَّلة.
>
> `> [لقطة: قائمة القبائل — تُضاف عند النشر]`

الإجراءات وما تستدعيه:
- **القائمة:** `GET /platform/tenants` (مع العدّادات).
- **الإيقاف/التفعيل:** `POST /platform/tenants/:id/suspend` و`.../activate`.

### حوار إنشاء قبيلة + أوّل مدير

> **وصف الشاشة:** حوار بعنوان «إنشاء قبيلة». حقلا **الاسم العربي** و**الاسم الإنجليزي**، وحقل **المُعرّف (slug)** (أحرف صغيرة وشرطات، مثل `bani-hilal`). ثمّ قسم **المدير الأوّل**: **الاسم الكامل**، **البريد الإلكتروني**، و**كلمة المرور** (≥ 12 حرفًا). زرّا **إلغاء** و**حفظ**. عند تكرار المُعرّف تظهر رسالة «المُعرّف مستخدَم».
>
> `> [لقطة: حوار إنشاء قبيلة — تُضاف عند النشر]`

- الإجراء: `POST /platform/tenants` بجسم `{ nameAr, nameEn, slug, admin: { email, fullName, password } }` — يُنشئ القبيلة ويعيّن أوّل **مدير قبيلة** لها.

---

## 3. إدارة الاشتراك (لكل قبيلة)

من عمود الإجراءات في جدول القبائل، زرّ **إدارة الاشتراك** يفتح حوارًا.

> **وصف الشاشة:** حوار باسم القبيلة، وأقسامه: **الاشتراك الحالي** (الخطّة، الحالة كوسم، تاريخ التفعيل، تاريخ الانتهاء مع شارة عدّ تنازليّ عند ≤ 30 يومًا). ثمّ **تعيين/تفعيل خطّة**: أربع بطاقات قابلة للاختيار — **مجّانيّة (≤ 500)**، **أساسيّة (≤ 5000)**، **احترافيّة (≤ 25000)**، **مؤسّسيّة (∞)** — وحقل **تاريخ انتهاء** وحقل **ملاحظة**، وتلميح «تفعيل يدويّ (تحويل بنكيّ)»، وزرّ **حفظ**. وأخيرًا **سجلّ التفعيلات** (تاريخ/الخطّة/بواسطة/ملاحظة).
>
> `> [لقطة: إدارة الاشتراك — تُضاف عند النشر]`

الإجراءات وما تستدعيه:
- **العرض:** `GET /platform/tenants/:id/subscription` ← الخطّة والحالة و`maxPersons` (null للمؤسّسيّة) و`currentPersons` الحيّ. القبيلة بلا اشتراك تُظهِر افتراضًا «مجّانيّة/نشِطة».
- **التعيين/التفعيل:** `PUT /platform/tenants/:id/subscription` بجسم `{ tier, expiresAt?, note? }` — يُسجَّل في سجلّ التفعيلات.
- **السجلّ:** `GET /platform/tenants/:id/subscription/activations`.
- **حدود الخطّة:** الخطط — مجّانيّة 500 / أساسيّة 5000 / احترافيّة 25000 / مؤسّسيّة غير محدودة. حارس مركزيّ يفرض السقف عند إنشاء الأشخاص/الاستيراد/النشر؛ فمثلًا قبيلة مجّانيّة عند 500 شخصًا يُرفَض الشخص الـ501 برسالة ترقية (`errors.subscription.plan_limit_reached`).

> **ملاحظة:** التفعيل يدويّ (تحويل بنكيّ) عبر تجريد `PaymentGateway` في النسخة الأولى؛ لا بوّابة دفع حقيقيّة.

---

## 4. الإحصاءات — `/statistics`

> **وصف الشاشة:** عنوان «الإحصاءات» مع تاريخ آخر تحديث، وزرّ **تحديث**. صفّ **مؤشّرات أداء (KPI)**: القبائل، النشِطة، الموقوفة، إجمالي الأشخاص، إجمالي المستخدمين. ثمّ قسمان: **القبائل حسب الخطّة** (أشرطة أفقيّة ملوّنة لكل خطّة مع العدد)، و**اشتراكات تقترب من الانتهاء** (جدول: اسم القبيلة كرابط + تاريخ الانتهاء). رسائل «لا بيانات» عند الفراغ.
>
> `> [لقطة: إحصاءات المنصّة — تُضاف عند النشر]`

- المصدر: `GET /platform/stats/dashboard` ← `{ tribes, activeTribes, suspendedTribes, totalPersons, totalUsers, byPlan[], expiringSoon[], refreshedAt }`.
- تُبنى الإحصاءات على **Materialized Views تُحدَّث كل ساعة**، ويمكن فرض تحديث فوريّ عبر `POST /platform/stats/refresh`. قائمة «تقترب من الانتهاء» تغذّي أيضًا شارات الصفوف في جدول القبائل.
