# أدلّة المستخدم لمنصّة TFTSP

منصّة **TFTSP** (منصّة شجرة العائلة والقبيلة كخدمة) هي نظام متعدّد المستأجرين (Multi-Tenant) يتألّف من ثلاثة تطبيقات:

| التطبيق | الاسم التقني | الجمهور | الوصف |
|---|---|---|---|
| **لوحة إدارة القبيلة** | `admin-web` (Angular) | مدير القبيلة، نائب المدير، مدير الفرع، المُراجع، المُساهم، المُشاهد | إدارة الأشخاص والوحدات القبلية والشجرة والموافقات والإعدادات والاستيراد والرؤية والسمعة. |
| **وحدة تحكّم المنصّة** | `platform-web` (Angular) | المشرف الأعلى (SuperAdmin) / مشرف المنصّة | إدارة القبائل (المستأجرين) والاشتراكات ولوحات الإحصاءات على مستوى المنصّة. |
| **تطبيق الأعضاء للجوال** | `mobile` (Flutter) | الأعضاء والمُساهمون والزوّار | تصفّح الشجرة، بطاقة الشخص، إنشاء المساهمات، الإشعارات، وطلب المشاهدة. |

> جميع قراءات بيانات الأشخاص في التطبيقات الثلاثة تمرّ عبر **مُحلِّل الرؤية (Visibility Resolver)** المركزي، فلا يصل أيّ حقل محجوب إلى الواجهة إطلاقًا.

## الأدوار في النظام

| الدور | الاسم التقني | الصلاحية الأساسية |
|---|---|---|
| المشرف الأعلى | `super_admin` | إدارة المنصّة كاملةً (القبائل + الاشتراكات + الإحصاءات). |
| مشرف المنصّة | `platform_admin` | صلاحيات منصّة (تشغيليّة). |
| مدير القبيلة | `tribe_admin` | التحكّم الكامل داخل قبيلته (كتابة مباشرة + كل الإعدادات). |
| نائب المدير | `deputy_admin` | كتابة مباشرة على الأشخاص والوحدات كالمدير (دون بعض الإعدادات الحصريّة للمدير). |
| مدير الفرع | `branch_admin` | كتابة مباشرة ضمن نطاق فرعه. |
| المُراجع | `reviewer` | مراجعة طلبات التغيير (موافقة/رفض/طلب تعديل). |
| المُساهم | `contributor` | اقتراح تغييرات عبر طلبات التغيير + سمعة. |
| المُشاهد | `viewer` | قراءة فقط، مع اقتراح محدود عند تفعيل القبيلة لذلك. |
| الضيف | `guest` | وصول عام محدود / طلب مشاهدة. |

**الأدوار ذات الكتابة المباشرة** (تُطبّق تعديلاتها فورًا دون موافقة): مدير القبيلة، نائب المدير، مدير الفرع. أمّا بقيّة الأدوار فتُحوَّل تعديلاتها إلى **طلبات تغيير (Change Requests)** تمرّ بسير الموافقات.
