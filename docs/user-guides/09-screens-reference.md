# مرجع الشاشات الكامل

مرجع فنّي لكل شاشات TFTSP الحقيقيّة، مُستخرَج من مسارات (routes) ومكوّنات التطبيقات الثلاثة. لكل شاشة: التطبيق، المسار، الأدوار، الغرض، العناصر الرئيسة (بمثابة **وصف الشاشة النصّي** بدل اللقطة)، والإجراءات وما تستدعيه من API.

- **التطبيقات:** `admin-web` = لوحة إدارة القبيلة · `platform-web` = وحدة تحكّم المنصّة · `mobile` = تطبيق الأعضاء (Flutter).
- **إجمالي الشاشات الموثّقة: 52** (31 في admin-web، 7 في platform-web، 14 في mobile — شاملةً القوقعات والحوارات والخطوات).
- **بدل اللقطات:** يلي كلَّ جدول قائمةُ عناصر نائبة `> [لقطة: <الشاشة> — تُضاف عند النشر]`.

## ملاحظات عامّة على الشاشات

- **مُحلِّل الرؤية:** كل قراءة شخص (قائمة/تفصيل/بحث/شجرة/أسلاف/أحفاد) تمرّ عبره؛ الحقول المحجوبة تُحذَف، وخارج النطاق يعود **404**.
- **الكتابة المباشرة مقابل الاقتراح:** الأدوار الإدارية تكتب مباشرةً؛ غيرها يُنشئ طلبات تغيير — وهذا يُبدّل عناوين الأزرار («حفظ» ↔ «إرسال للمراجعة») ويُظهِر قائمة **نوع المساهمة**.
- **التوطين والاتجاه:** كل الشاشات تدعم AR (RTL) / EN (LTR) بتبديل فوريّ دون إعادة تحميل، وتتبع ألوان القبيلة.
