# حزمة مواصفات منصة TFTSP

> **TFTSP — Tribal & Family Tree SaaS Platform** — منصة SaaS متعددة المستأجرين لإدارة أشجار العائلات والقبائل العربية، ببنية واحدة تخدم آلاف القبائل مع عزل صارم للبيانات ودعم كامل للعربية (RTL) والإنجليزية (LTR).

هذا المجلد (`docs/specification/`) هو **المرجع الموحّد** الذي يصف النظام كما بُني فعليًا. كل عبارة فيه مستمدة من الكود الحقيقي: مخطط قاعدة البيانات `apps/api/prisma/schema.prisma`، عقود الـ API المُجمَّدة `docs/API_CONTRACT.M1..M5.md`، مصفوفة الصلاحيات `apps/api/src/common/rbac/permissions.ts`، ومتحكّمات الوحدات في `apps/api/src/modules/**`.

## نبذة عن النظام

النظام أربعة مكوّنات تتقاسم واجهة برمجية واحدة (`/api/v1`):

| # | المكوّن | المسار | الوصف |
|---|---|---|---|
| ① | Backend API | `apps/api` | خادم NestJS واحد (Prisma + PostgreSQL 16 مع RLS، Redis، BullMQ، Socket.IO، MinIO) — مصدر الحقيقة والأمن لكل العملاء |
| ② | Tribe Admin Panel | `apps/admin-web` | لوحة إدارة القبيلة (Angular) لأدوار القبيلة الإدارية |
| ③ | SaaS Platform Panel | `apps/platform-web` | لوحة إدارة المنصة (Angular) للـ Super/Platform Admin على نطاق منفصل |
| ④ | Members App | `apps/mobile` | تطبيق الأعضاء (Flutter — Android + iOS بكود واحد) |

تُشارَك الأنواع عبر حزمة `packages/shared-types`.

## حالة التنفيذ

النظام **مكتمل بالكامل**: المراحل **M1 → M2 → M2.5 → M3 → M4 → M5** مبنية ومدموجة، وكل مرحلة أُغلقت بمعايير قبولها قبل الانتقال إلى التالية.

| المرحلة | النطاق | الحالة |
|---|---|---|
| **M1** | النواة: العزل متعدد المستأجرين (RLS) + الهوية (JWT/Argon2id) + الأشخاص + الزيجات (Union) + الأنساب (Closure Table) + الوحدات القبلية + Audit | ✅ مكتملة |
| **M2** | طلبات التغيير (Change Requests) + سير الموافقات + التنبيهات (in-app WebSocket + بريد MJML) | ✅ مكتملة |
| **M2.5** | الاستيراد الجماعي (xlsx/CSV، Streaming، Staging، Rollback) | ✅ مكتملة |
| **M3** | الرؤية والخصوصية (Visibility Resolver) + طلبات المشاهدة | ✅ مكتملة |
| **M4** | العرض المتقدم + التصدير (PDF/PNG/xlsx/CSV) + الاشتراكات + الوثائق + المساهمات والسمعة | ✅ مكتملة |
| **M5** | تطبيق الأعضاء (Flutter) + تسجيل الأجهزة + دفع FCM | ✅ مكتملة |

**التحقّق:** يفرض CI (`.github/workflows/ci.yml`) الـ lint والاختبارات والبناء على كل PR — بما فيه **اختبار العزل الشامل** (مستخدم من قبيلة يحاول الوصول إلى موارد قبيلة أخرى ⇐ `403/404`)، واختبارات سلامة النسب (استحالة حلقات النسب)، وصحّة صيانة Closure Table، إضافةً إلى `flutter analyze && flutter test` للموبايل.

## القرارات المرجعية

القرارات المعمارية المحسومة موثّقة في القسم 2 من البرومبت الأصلي، وحلول الغموض في `DECISIONS.md` (المعرّفات `D-001…D-5xx`). عقود الـ API مجمّدة كنقطة تزامن بين الفرق في `docs/API_CONTRACT.M1..M5.md`.
